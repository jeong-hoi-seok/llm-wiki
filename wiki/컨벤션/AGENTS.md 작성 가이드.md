# AGENTS.md 작성 가이드

신규 프로젝트에서 AI가 위키를 실제로 읽고 적용하는지 검증하는 구조를 포함한 AGENTS.md 작성 기준.
상위: [[index]]

---

## 핵심 원칙

AGENTS.md에는 네 가지가 반드시 있어야 한다:
1. **파일명 고정** — 파일명은 반드시 대문자 `AGENTS.md`로 쓴다. `agent.md`, `agents.md`, `Agent.md`처럼 소문자로 쓰지 않는다.
2. **명령형 읽기** — "참조한다" 아닌 "직접 열어서 읽는다"
3. **검증 장치** — AI가 읽었는지 추적할 수 있는 규칙
4. **심볼릭링크 연동** — 위키 실체는 프로젝트 밖 `$HOME/.llm-wiki` 한 곳에서 관리하고, 프로젝트에는 그곳을 가리키는 `.llm-wiki` 심볼릭링크만 둔다. 링크가 없으면 원격에서 가져와 세팅한다.

---

## 공통 위키 위치 / 연동 방식

위키 **실체는 홈 디렉터리 `$HOME/.llm-wiki` 한 곳에만** 둔다. 어떤 코드 프로젝트도 복제본을 두지 않는다. 프로젝트 루트에는 그 실체를 가리키는 `.llm-wiki` **심볼릭링크**(포인터)를 만들고, 항상 프로젝트 내부 상대경로 `./.llm-wiki/...`로 읽는다.

- 서브모듈과 달리 실체 복제본이 프로젝트에 들어오지 않는다(포인터만, 디스크 0).
- 상대경로 `./.llm-wiki/...`만 읽기 도구에 넘긴다. `$HOME`·`~`·절대경로를 그대로 넘기지 않는다 — 링크가 머신차를 흡수하므로 AGENTS.md에 경로를 하드코딩할 필요가 없다.

**git 올리는 대상 구분 (혼동 주의):**

| 대상 | git |
|---|---|
| 본진 `~/.llm-wiki` repo (실체이자 source) | GitHub에 **정상 커밋·push 한다.** |
| 소비 프로젝트 안의 `.llm-wiki` 심볼릭링크 (포인터) | **절대 올리지 않는다.** 소비 프로젝트 `.gitignore`에 `.llm-wiki` 선언 **필수**, 머신마다 재생성. |

심볼릭링크는 절대경로로 만들어져 다른 머신에서 깨진다. 그래서 커밋하지 않고 머신마다 부트스트랩으로 재생성한다.

---

## 부트스트랩 절차 (작업 시작 전 필수)

소비 프로젝트에서 작업을 시작하기 전에 위키를 연결한다.

1. **위키 실체 확보** (머신당 1회). 홈에 없으면 원격 `wiki/index.md`를 먼저 읽고 clone, 있으면 pull.
   ```bash
   [ -d "$HOME/.llm-wiki/wiki" ] \
     && git -C "$HOME/.llm-wiki" pull --ff-only \
     || git clone https://github.com/jeong-hoi-seok/llm-wiki.git "$HOME/.llm-wiki"
   ```
2. **심볼릭링크 생성** (프로젝트 루트에서).
   - macOS / Linux / WSL: `ln -sf "$HOME/.llm-wiki" ./.llm-wiki`
   - Windows PowerShell(관리자): `New-Item -Force -ItemType SymbolicLink -Path .\.llm-wiki -Target "$HOME\.llm-wiki"`
3. **`.gitignore`에 `.llm-wiki` 선언** (필수). 링크는 절대 커밋하지 않는다.
4. **검증**: `./.llm-wiki/wiki/index.md`가 실제로 열리는지 확인한다(링크 존재만으로 갈음하지 않는다).

clone/pull/링크/검증 중 하나라도 실패하면(네트워크·권한) 사실을 말하고 중단한다.

## 적용 방식

> **기존 `AGENTS.md`가 있는 경우** — 아래 `Wiki Baseline` 블록만 기존 파일에 삽입한다. 기존 내용은 건드리지 않는다.
>
> **`AGENTS.md`가 없는 경우** — 아래 전체 템플릿으로 새로 작성한다.
>
> **파일명 주의** — 에이전트가 소문자 파일명을 놓치는 경우가 있으므로 프로젝트 root 파일명은 반드시 `AGENTS.md`로 만든다.

소비 프로젝트 `AGENTS.md`의 `Wiki Baseline`은 **짧게** 유지한다. 연동·셋업·gitignore·읽기/응답 규칙 상세는 모두 이 가이드에 있으므로, 소비 쪽에서는 반복하지 않고 이 문서를 가리키기만 한다.

### 삽입 전용 블록 (기존 파일에 추가할 내용)

````md
## Wiki Baseline

공통 위키(`~/.llm-wiki`)를 참조해 작업한다.

- **로컬에 위키가 있으면** 심볼릭링크로 `./.llm-wiki/wiki/index.md`를 직접 열어 읽는다.
- **없으면** 원격 `https://github.com/jeong-hoi-seok/llm-wiki`의 `wiki/index.md`를 먼저 읽고, clone → 심볼릭링크로 세팅한 뒤 진행한다.

연동·셋업·`.gitignore`·읽기/응답 규칙 상세는 위키 `wiki/index.md` 또는 `wiki/컨벤션/AGENTS.md 작성 가이드.md`를 따른다.

작업 결과물 하단에 참고한 위키 문서를 반드시 명시한다(없으면 위키 미열람으로 간주):
<!-- wiki-ref: 컨벤션/커밋 컨벤션.md, 개발원칙/가독성.md -->
````

### 전체 템플릿

````md
# AGENTS.md

모든 AI 에이전트가 이 프로젝트에서 따를 공통 지침.

## Project Identity

```yaml
platform: web   # web | app
```

| 값 | 스택 예시 |
|---|---|
| `web` | Next.js, React (DOM), Tailwind CSS |
| `app` | React Native, Expo, NativeWind |

### 프로젝트 요약

- **이름**: {프로젝트명}
- **스택**: {스택 목록}

---

## Wiki Baseline

(위 "삽입 전용 블록"과 동일하게 넣는다. `platform` 값에 맞는 태그 문서 `[공통]`+해당 플랫폼만 추가로 읽는다.)
````

## 읽기 규칙

> 아래 파일을 반드시 직접 열어서 읽는다. 경로만 인지하는 것으로 갈음하지 않는다.

1. 심볼릭링크를 통해 `./.llm-wiki/wiki/index.md`를 직접 연다. (`$HOME`·`~`·절대경로 금지, 상대경로 `./.llm-wiki/...`만)
2. 프로젝트 `platform` 값에 맞는 태그 문서만 추가로 읽는다(`[공통]` + 해당 플랫폼).
3. 작업 주제와 연결된 문서를 index에서 찾아 추가로 읽는다.

## 검증 방법

AGENTS.md를 작성한 뒤 아래 방법으로 AI가 실제로 위키를 읽었는지 확인한다.

### 방법 1. wiki-ref 확인 (수동)

응답 하단에 `<!-- wiki-ref: ... -->` 가 있는지 확인.
- 있음 → 위키를 읽고 경로를 추적했음
- 없음 → 위키 미참조 (AGENTS.md 규칙 미적용)

### 방법 2. 카나리아 규칙 (자동)

위키 문서에 **학습 데이터에 없는 독특한 규칙**을 심어둔다.
AI가 해당 규칙을 따르면 위키를 읽은 것으로 판단.

현재 심어진 카나리아:
- `컨벤션/커밋 컨벤션.md` — subject 반드시 한국어, 명사형 어미
- `컨벤션/MR PR 작성 가이드.md` — 브랜치 확인 없이 본문 작성 금지

새 카나리아 추가 방법:
1. 위키 문서에 비표준적이지만 합리적인 규칙 추가
2. 해당 규칙을 따르는지 테스트 프롬프트로 확인

### 방법 3. 툴콜 로그 확인 (Cursor 전용)

Cursor 채팅창에서 AI가 실제로 파일을 열었는지 확인.
`Read` 툴콜에 위키 경로가 찍히면 읽은 것.

## 흔한 실수

| 실수 | 결과 | 수정 |
|---|---|---|
| "참조한다"로 작성 | AI가 경로만 인지, 파일 미열람 | "직접 열어서 읽는다"로 변경 |
| wiki-ref 규칙 없음 | 읽었는지 추적 불가 | 응답 규칙 한 줄 추가 |
| platform 미선언 | 앱/웹 필터 미적용 | `platform: web` or `app` 명시 |
| 절대경로 하드코딩 / 읽기 도구에 `$HOME`·`~` 전달 | 다른 머신에서 깨짐, 경로 못 찾음 | 심볼릭링크 통해 상대경로 `./.llm-wiki/...`로 읽는다 |
| 심볼릭링크를 git에 커밋 | 다른 머신에서 깨진 링크가 따라옴 | 소비 프로젝트 `.gitignore`에 `.llm-wiki` 선언, 머신마다 재생성 |
| 위키 미존재 시 그냥 진행 | 빈 경로 읽고 위키 무시 | 없으면 원격 index.md 읽고 clone → 심볼릭링크 세팅 |
| Wiki Baseline에 부트스트랩 전문 복붙 | 가이드와 중복·표류 | 소비 쪽은 짧게, 상세는 이 가이드를 가리키기 |
