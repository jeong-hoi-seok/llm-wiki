# 위키 버전 관리

## 한 줄 요약

이 위키의 버전은 `wiki/index.md` frontmatter의 `version` 값으로 관리한다. 현재 시작 버전은 `0.3.0`이다.

## 왜 `0.3.0`으로 시작하나

`0.1.0`은 초기 스캐폴드 수준에 가깝다. 현재 위키는 이미 `raw/`, `wiki/`, `context/`, `log/`, `AGENTS.md` 운영 규칙과 주요 프론트엔드 문서를 갖췄다.

다만 아직 1.0은 아니다.

- 팀 공유용 경로 resolver는 문서화됐지만 실제 팀 적용 검증이 없다.
- 끊어진 링크·고립 문서·출처 검증 자동화가 없다.
- 기술 범위가 Next.js·Expo·FSD·테스트 일부에 치우쳐 있다.
- 버전 정책도 이제 막 생겼다.

그래서 현재 상태는 **사용 가능한 초안 기준선**으로 보고 `0.3.0`에서 시작한다.

## 버전 형식

형식은 SemVer를 빌려온다.

```txt
MAJOR.MINOR.PATCH
```

1.0 전까지는 `0.MINOR.PATCH`를 사용한다. `1.0.0`은 팀 공유 기준선이 충분히 안정됐을 때만 올린다.

## 올리는 기준

여러 변경이 한 작업에 섞이면 가장 큰 기준 하나만 적용한다.

| 변경 | 예시 | 버전 |
|---|---|---|
| 운영 방식 변경 | AGENTS 흐름 변경, wiki 폴더 역할 변경, 경로 resolver 도입 | minor |
| 새 컨벤션 추가 | 버전 관리, 커밋, PR, 프로젝트 문서화 규칙 추가 | minor |
| 새 대형 원문 ingest | 새 raw 묶음 처리 + 관련 개념 여러 문서 생성 | minor |
| 새 카테고리 추가 | `테스트/`, `최적화/`, `기술비교/` 같은 축 신설 | minor |
| 기존 판단 기준 변경 | 기술결정 결론 변경, 기존 컨벤션 의미 변경 | minor |
| 문서 이동·분류 변경 | 템플릿 문서를 컨벤션으로 이동, index 구조 변경 | minor |
| 기존 문서 보강 | 설명 추가, 예시 보강, 출처 보강 | patch |
| 오탈자·표현 수정 | 의미 변화 없는 문장 정리 | patch |
| 링크·index·log 보정 | 누락 링크 추가, 오래된 목차 갱신 | patch |
| 질문 답변 | 파일 변경 없는 답변 | 변경 없음 |

## 1.0.0 기준

아래 조건을 만족하면 `1.0.0` 검토 가능.

- `AGENTS.md`와 주요 프로젝트용 `AGENTS.md 작성 가이드`가 팀 공유 방식까지 포함한다.
- `wiki/index.md`가 핵심 문서를 빠짐없이 가리킨다.
- 주요 문서가 출처, 플랫폼, 상태를 가진다.
- 링크 lint 기준이 정해져 있고 큰 깨짐이 없다.
- 공통 지식과 프로젝트 맥락 문서 역할이 명확히 분리돼 있다.

## 작업 절차

1. 변경 범위를 확인한다.
2. 이 문서의 기준으로 `minor` 또는 `patch`를 고른다.
3. `wiki/index.md` frontmatter의 `version`과 `updated`를 갱신한다.
4. `wiki/log/YYYY-MM-DD.md`에 이전 버전, 새 버전, bump 이유를 남긴다.
5. `wiki/log.md` 날짜별 요약도 갱신한다.
