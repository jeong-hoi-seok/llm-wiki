# 위키 개요

이 위키가 무엇이고 어떻게 자라는지에 대한 개요. 목차는 [[index]].

## 무엇인가
프론트엔드(웹·앱) 개발 지식을 LLM이 읽고·요약하고·연결해 계속 키우는 Markdown 위키.
Andrej Karpathy의 LLM Wiki 방식 기반.

## 구조
```
raw/    = 원본 책장 (사람이 넣음, LLM 읽기 전용)
wiki/   = LLM이 만든 백과사전 (기술개념·summaries·기술결정·…)
wiki/기술개념/web = 브라우저·DOM·Next.js·Tailwind·웹 번들러 기술
wiki/기술개념/app = React Native·Expo·NativeWind·네이티브 런타임 기술
index.md = 전체 목차 + 위키 버전
log.md   = 작업 이력 색인
log/     = 일자별 작업 이력
CLAUDE.md / AGENTS.md = AI 운영 매뉴얼
```

## 운영 루프
```
Ingest → Query → Lint  (반복)
```
- **Ingest**: `raw/` 새 자료 → `summaries/` 요약 + `기술개념/` 개념 생성·연결 → `index`·`log/YYYY-MM-DD` 갱신
- **Query**: `index` 읽고 → 관련 문서로 근거 답변
- **Lint**: 끊어진 링크·고립·중복·무출처 점검 → 수정 또는 `log/YYYY-MM-DD` 기록
- **Versioning**: `wiki/` 변경 → `index.version` 갱신 → `log/YYYY-MM-DD`에 이유 기록

## 사용 예
```
raw/feature-sliced-design/kr/docs/를 ingest 해줘. AGENTS.md 규칙 따르고 index·log 갱신.
내 위키 기준으로 Next.js App Router에서 FSD 폴더 구조 잡는 법 설명해줘.
wiki/를 lint 해줘 (끊어진 링크·고립·중복·무출처·index 누락).
```
