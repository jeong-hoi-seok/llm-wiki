# 우아한형제들 단위 테스트 시리즈 요약

## 한 줄 요약
우아한형제들 단위 테스트 2부작은 React 코드에서 단위 테스트를 “작은 코드 검증”이 아니라 **외부 동작을 명세로 남기고 유지보수 비용을 낮추는 장치**로 설명한다.

## 핵심 내용
- 단위 테스트는 특정 모듈이나 컴포넌트가 의도한 기능을 수행하는지 확인한다.
- 좋은 단위 테스트는 F.I.R.S.T 원칙을 따른다. 빠르고, 독립적이고, 반복 가능하고, 스스로 성공/실패를 판단할 수 있어야 한다.
- 테스트 코드에서는 중복 제거보다 설명력과 의미 전달이 더 중요할 수 있다(DAMP 원칙).
- 구현 세부사항보다 사용자가 볼 수 있는 결과를 검증해야 한다(내부 구현 비검증).
- React Testing Library에서는 `screen.getByRole`, `userEvent`처럼 사용자 관점 API를 우선한다.
- 테스트명은 구현이 아니라 정책과 명세를 설명해야 한다.
- Part 2는 실제 컴포넌트/훅 테스트를 다루며, loading 대기, input 글자 수 제한, 조건부 렌더링, 사용자 입력 후 callback 호출 같은 케이스를 코드로 보여준다.
- 테스트 코드도 프로덕션 코드처럼 유지보수해야 하며, 작성 비용과 장애 위험도를 함께 고려한다.
