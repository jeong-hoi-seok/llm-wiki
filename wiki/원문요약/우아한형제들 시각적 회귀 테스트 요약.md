# 우아한형제들 시각적 회귀 테스트 요약

## 한 줄 요약
우아한형제들 시각적 회귀 테스트 글은 디자인 시스템에서 단위 테스트로 잡기 어려운 UI 스타일 변경을 Playwright 스크린샷 비교로 검출하는 사례다.

## 핵심 내용
- 단위 테스트 커버리지가 높아도 시각적 변경은 놓칠 수 있다.
- 디자인 시스템처럼 컴포넌트 간 의존이 많은 환경에서는 한 컴포넌트 변경이 다른 컴포넌트 UI에 영향을 줄 수 있다.
- Storybook을 테스트베드로 사용하면 이미 작성된 상태별 스토리를 시각 테스트 자산으로 재사용할 수 있다.
- Playwright는 스크린샷 비교, HTML 리포트, E2E 확장 가능성을 제공한다.
- 스크린샷 테스트는 렌더링 완료, 타이머, 애니메이션, 환경 차이 같은 변동성을 제어해야 한다.
- 원문 사례의 디자인 시스템은 115개 UI 컴포넌트를 제공하고, 컴포넌트 단위 테스트 커버리지 91%를 유지하고 있었다. 그럼에도 스타일 변경과 의존 컴포넌트 영향은 별도 시각 테스트가 필요했다.
- 테스트베드 후보는 Storybook, 테스트 앱, 공식 문서 사이트였고, Storybook이 iframe으로 컴포넌트만 렌더링할 수 있으며 기존 Story를 재사용할 수 있어 선택되었다.
- 렌더링 완료 시점은 CSR 환경에서 `load`, `domContentLoaded`만으로 보장되지 않아 MutationObserver를 활용했다.
- 자동 슬라이드처럼 시간 영향을 받는 요소는 스크린샷 테스트에서 통제해야 한다.
- CI에서 Playwright Docker image로 테스트를 실행하고 HTML 리포트를 MR에서 확인할 수 있게 구성했다.
