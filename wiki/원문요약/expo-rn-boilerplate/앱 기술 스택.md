# 앱 기술 스택 (RN + Expo)

React Native + Expo 보일러플레이트 기준 스택·전제. 상위: [[index]]

> 기준: Expo SDK 54 boilerplate. 핀 버전은 고정 — 의존성 추가·업데이트 시 SDK 54 호환성 먼저 확인. (확인: 2026-06)
> **왜 하필 SDK 54.0.35인가** (Expo Go·App Store·레거시 아키텍처) → [[Expo SDK 54 버전 고정]]

## 스택

| 영역 | 선택 |
|---|---|
| 프레임워크 | Expo `54.0.35` · React Native `0.81.5` |
| UI | React `19.1.0` · `@types/react` `~19.1.10` |
| 언어 | TypeScript (strict) |
| 스타일 | NativeWind · CVA · `cn` → [[NativeWind]] |
| 상태 | Zustand |
| 린트/포맷 | Biome |
| 패키지 매니저 | **pnpm** (`npm`/`yarn`/`bun` 사용 안 함) |

## 전제 규칙
- 위 핀 버전 고정. `pnpm add`·`pnpm exec`만 사용.
- safe area(`useSafeAreaInsets`, `SafeAreaView`)는 화면 레이아웃 **기본 전제**.
- 라우트는 루트 `app/`이 아니라 **`src/app/`** ([Top-level src directory](https://docs.expo.dev/router/reference/src-directory/), SDK 54+ 자동 인식).
- alias `@/*` → `./src/*`, 정적 에셋은 루트 `assets/` → `@/assets/*`.

## 아키텍처 (FSD 간소화)
```txt
src/
  app/        # Expo Router 라우트
  features/   # 기능 slice (ui / model / api / lib / index.ts)
  entities/   # 도메인 엔티티
  shared/     # ui / lib / config (세그먼트별 index.ts)
assets/       # 정적 에셋 (@/assets/*)
styles/       # global.css (Tailwind 진입점)
```
- 일반 FSD 원리 → [[FSD 폴더구조]]
- slice 외부는 `index.ts` public API만 import. 같은 계층 slice 직접 import 금지.

## 작업 원칙
1. UI 스타일은 NativeWind `className` 우선. variant UI는 CVA + `cn`.
2. RN·Expo API 동작은 추측 말고 공식 문서 → [[공식 문서 참조]]
3. 기존 패턴 유지, 요청 범위만 수정. 새 파일은 kebab-case → [[네이밍 컨벤션]]
4. 검증 가능하면 `pnpm lint` / `pnpm typecheck` 후 보고.
