## 핵심 요약
접근성(접근성, A11y) = 모든 사용자가 쉽게 웹을 쓰게 하는 기본 원칙. 프론트엔드 개발자가 HTML 구조·인터랙션을 만들므로 **접근성의 출발점**. 4대 원칙으로 정리. → [[접근성]]

## 주요 주장
- **장애인**: 스크린 리더가 `role`·`label`·`alt` 없으면 정보·상호작용 놓침. `<div onclick>`은 버튼으로 인식 안 됨.
- **일반 사용자에게도 유용**: `<a>`로 새 창 열기/링크 복사, `<form>`으로 Enter 제출, 키보드(Tab) 탐색 — 시맨틱 마크업이 기본 동작 제공.
- **개발자 효용**: 접근성 잘 지키면 `getByRole`로 테스트 요소 특정 쉬움 (querySelector·testId보다 견고).

## 4대 원칙 (→ [[접근성]] 본문)
1. 올바른 구조 (버튼 안 버튼 금지, 테이블 행 링크)
2. 의미 정확히 전달 (인터랙티브 요소 이름)
3. 예측 가능한 인터랙션 (가짜 버튼 금지, input은 form)
4. 시각 정보에만 의존하지 않기 (대체 텍스트)

## 실무에서 가져갈 점
- 스크린 리더 3요소 = 역할(Role)·레이블(Label)·상태(State).
- eslint로 코드 단계에서 접근성 문제 사전 탐지.
