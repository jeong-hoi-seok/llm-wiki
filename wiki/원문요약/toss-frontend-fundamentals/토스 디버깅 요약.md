# 토스 디버깅 요약

## 원문 정보
- 제목: 디버깅 — 버그 패턴과 디버깅 접근법
- 저자: 토스 (Frontend Fundamentals)
- URL: https://frontend-fundamentals.com `[검증 필요]`
- 원본: `raw/toss/frontend-fundamentals/debug/`

## 핵심 요약
디버깅 = 예상대로 동작 안 할 때 원인을 찾아 해결하는 과정. **감에 의존하지 않고 체계적으로** — 진단 → 수정 → 예방 3단계. → [[디버깅]]

## 주요 주장
- **진단**: 에러 메시지에 압도되지 말고 원인을 구체적으로 좁히기.
- **수정**: 증상 제거가 아니라 **근본 원인** 수정. 데드코드 제거, 순수 함수 분리(테스트 용이).
- **예방**: 버그 수정은 끝이 아니라 더 나은 시스템·팀의 출발점. 에러 로그 상세 기록, 버그 리포트, 팀 공유.

## 실무에서 가져갈 점
- 데드코드(미사용 유틸·CSS·끝난 A/B 실험)는 디버깅을 방해 → 기능 제거 시점에 함께 삭제.
- 비즈니스 로직을 순수 함수로 분리 → UI 없이 테스트 가능.
- 에러 로그에 사용자 상태(버전·기기·OS)·재현 조건 기록.
