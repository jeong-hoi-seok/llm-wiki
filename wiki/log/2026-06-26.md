# 2026-06-26

## update | AGENTS.md 작성 가이드 위키 부트스트랩 방식 전환

- 문제: 기존 가이드는 공통 위키가 이미 어딘가 clone돼 있다고 전제하고 `FRONTEND_LLM_WIKI_ROOT` 환경변수 → `$HOME/.frontend-llm-wiki` symlink 순서로 위치를 찾았음. 머신마다 경로가 제각각이고 위키가 아예 없는 머신은 대응 안 됨
- [[AGENTS.md 작성 가이드]]를 `$HOME/.llm-wiki` 단일 위치 부트스트랩 방식으로 변경:
  - 셸별로 홈 절대경로를 먼저 구한다(bash/zsh/Git Bash/WSL `echo "$HOME"`, PowerShell `echo $HOME`, cmd `echo %USERPROFILE%`) → 위키 루트 = `HOME_DIR + /.llm-wiki`
  - 위키 루트에 `wiki/index.md`가 없으면 머신당 1회 `git clone`, 있으면 `git pull --ff-only`로 최신화
  - clone/pull 실패 시(네트워크·권한) 사실을 말하고 중단
  - 읽기 순서: 푼 절대경로로 `<위키 루트>/wiki/index.md`를 직접 열고, `$HOME`·`~`를 그대로 읽기 도구에 넘기지 않음
- 핵심 원칙 4번을 "경로 간접화(환경변수+symlink)"에서 "위키 부트스트랩(`$HOME/.llm-wiki` 단일 관리·없으면 clone·절대경로)"로 교체
- 삽입 전용 블록과 전체 템플릿의 Wiki Baseline을 동일하게 갱신, clone URL은 `https://github.com/jeong-hoi-seok/llm-wiki.git`로 박음
- 흔한 실수 표 갱신: 절대경로 하드코딩, 위키 미존재 시 그냥 진행, 읽기 도구에 `$HOME`·`~` 그대로 전달 케이스 추가
- [[index]] AGENTS.md 가이드 설명을 "공통 위키 경로 resolver" → "`$HOME/.llm-wiki` 위키 부트스트랩"으로 수정
- 버전 변경: `0.6.0` → `0.6.1`
- bump 기준: 단일 컨벤션 문서의 기존 기준 조정(스코프 안)이고 이 위키 자체의 ingest/query 작동 방식은 그대로이므로 patch 기준
