# 우아한형제들 통합 테스트 요약

## 한 줄 요약
우아한형제들 통합 테스트 글은 React 화면에서 함수, 컴포넌트, 서버 응답, 사용자 인터랙션이 함께 맞물리는 흐름을 테스트하는 기준을 설명한다.

## 핵심 내용
- 단위 테스트만으로 애플리케이션 전체 동작을 확신하기 어렵다.
- 통합 테스트는 여러 컴포넌트와 API 응답이 연결된 사용자 흐름을 검증한다.
- 약관 API를 모킹하고 필수 약관 체크 후 주문 버튼이 활성화되는지 확인하는 예시처럼, 사용자의 실제 행동에 가까운 시나리오가 핵심이다.
- 고수준 테스트가 많아질수록 모킹 수, 실행 시간, 유지보수 비용도 커진다.
- 따라서 단위·통합 선택 기준을 두고 테스트 범위를 나누어야 한다([[통합 테스트]]).
- 포인트, 쿠폰, 결제수단처럼 서로 영향을 주는 영역은 단위 테스트만으로 놓치는 흐름이 생길 수 있다.
- 그러나 주문서 페이지 전체를 항상 렌더링하면 외부 시스템과 mock이 과도하게 늘어난다.
- 좋은 통합 테스트는 “사용자 경험과 가까운 신뢰도”와 “운영 가능한 테스트 비용” 사이에서 경계를 잡는다.
