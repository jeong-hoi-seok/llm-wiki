# 트리 쉐이킹

## 한 줄 요약
프로젝트에서 **사용되지 않는 코드(Dead Code)를 번들에서 제거**하는 최적화. 나무를 흔들어 죽은 잎을 떨구듯. [[번들링]] 최적화 기법.

## 핵심
- 라이브러리에서 함수 하나만 써도 전체가 번들에 들어가는 낭비를 방지 → 번들 크기↓ → 로딩↑.
- **정적 분석(Static Analysis) 기반** → 코드 실행 전 구조 분석 → **빌드 타임**에 적용.

## 왜 ESM이 유리한가
| 모듈 시스템 | 트리셰이킹 |
|---|---|
| **ESM** (`import`/`export`) | 정적 구조 → 빌드 타임에 의존성 그래프 분석 가능 → **효과적** |
| **CJS** (`require`) | 동적 구조 → 분석 어려움 → 비효율 |
- 라이브러리·코드를 ESM으로 작성·배포하면 트리셰이킹이 잘 동작.
