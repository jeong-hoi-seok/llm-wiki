# 번들링

## 한 줄 요약
여러 파일(JS·CSS·이미지)을 하나/몇 개로 묶는 작업. 브라우저 요청 수를 줄여 로딩 속도를 높인다. 상위: [[index]]

## 왜 중요한가
파일을 기능별로 잘게 쪼개면 개발은 편하나, 브라우저가 파일 수만큼 요청 → 네트워크 비용·로딩 지연. 번들링이 이를 해결.
- **목적**: ① 요청 수 감소 ② 캐싱 최적화 ③ 개발 시 모듈화 유지 + 배포 시 성능 최적화.

## 번들 과정
```
여러 JS 파일 → 의존성(import) 분석 → 의존성 그래프 → 순서대로 하나로 묶기(번들)
```
시작점(entry, 예: index.js)부터 의존 파일을 추적.

## 번들러가 하는 일
번들러 = 묶기 + 변환 + 최적화 + 개발환경 제공.
1. **파일 묶기** — 요청 수↓, 모듈 시스템(ESM/CJS) 사용 가능.
2. **코드 변환(Transpiling)** — TS·JSX → 브라우저용 JS (Babel 등).
3. **최적화** — [[트리 쉐이킹]](미사용 코드 제거)·Minification(압축)·[[코드 스플리팅]]·Lazy Loading·이미지 최적화.
4. **개발환경** — [[HMR]](새로고침 없이 교체)·Source Map(번들↔원본 매핑, 디버깅).

## 대표 번들러
| 번들러 | 특징 |
|---|---|
| **Webpack** | 가장 널리 쓰임. 플러그인·로더 생태계 강력. 설정 복잡·속도 느린 편 |
| **Vite** | 빠른 개발 서버 (esbuild 기반 dev) |
| **ESBuild** | Go 기반 초고속 |
| **Rollup** | 라이브러리 제작 최적 |
